# Этап 03 — Доменная модель

> [!todo] Цель этапа
> Миграции и Eloquent-модели всех сущностей + factories/seeders с демо-данными.

## Задачи

- [x] Миграции: `users`, RBAC (Spatie), профили `managers`, `teachers`, `students`
- [x] `applicants` — ФИО, СНИЛС, паспорт, статус воронки, `manager_id`, `external_id`
- [x] `contracts` — номер, файл, статус, даты, `applicant_id`
- [x] `classrooms` — номер, корпус, вместимость
- [x] `schedule_slots` — дата/время, тип занятия, `teacher_id`, `classroom_id`
- [x] `work_sessions` — фактические часы по типам
- [x] `audit_logs`, `integration_logs`, `notifications`
- [x] Eloquent-модели с отношениями
- [ ] Factories для всех сущностей
- [ ] `DemoDataSeeder` — 10 абитуриентов, 3 педагога, 20 слотов, 5 студентов

## Ключевые таблицы

| Таблица | Назначение |
|---------|------------|
| `applicants` | Воронка приёма, СНИЛС, привязка к менеджеру |
| `contracts` | Договоры обучения, подписание |
| `schedule_slots` | Расписание: лекция / лаб / консультация |
| `work_sessions` | Учёт отработанных часов педагога |
| `students` | Группа, курс, связь с `user_id` |

## Статусы воронки абитуриента

`new` → `contacted` → `contract_draft` → `contract_signed` → `enrolled` / `rejected`

## Критерий завершения

> [!success] Готово, когда
> `php artisan migrate --seed` создаёт полную демо-базу; все модели имеют корректные отношения; smoke-тест CRUD через tinker проходит.
