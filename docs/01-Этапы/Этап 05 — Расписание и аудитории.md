# Этап 05 — Расписание и аудитории

> [!todo] Цель этапа
> CRUD слотов расписания, управление аудиториями, детекция конфликтов.

## Задачи

- [ ] `ScheduleSlotController` — CRUD с валидацией дат и типов занятий
- [ ] `ClassroomController` — создание аудиторий, проверка доступности
- [ ] `ScheduleConflictService` — конфликты педагога и аудитории
- [ ] `GET /api/classrooms/availability` — свободные слоты
- [ ] Frontend: календарь/неделя для педагога (`pages/teacher/`)

## Типы занятий

- `lecture` — лекция
- `lab` — лабораторная
- `consultation` — консультация

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET/POST | `/api/schedule-slots` | CRUD расписания |
| GET/POST | `/api/classrooms` | Аудитории |
| GET | `/api/classrooms/availability` | Свободные слоты |

## Критерий завершения

> [!success] Готово, когда
> Педагог создаёт слот; система отклоняет пересечение по аудитории или педагогу; UI показывает расписание на неделю.
