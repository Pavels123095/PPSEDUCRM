# Этап 04 — Модуль менеджеров

> [!todo] Цель этапа
> API и UI: воронка абитуриентов, создание и подписание договоров.

## Задачи

- [ ] `ApplicantController` — CRUD + фильтры по статусу/менеджеру
- [ ] `ContractController` — создание, загрузка файла, фиксация подписания
- [ ] `PATCH /api/applicants/{id}/status` — смена этапа воронки
- [ ] `GET /api/managers/dashboard` — сводка по воронке
- [ ] Frontend: dashboard, список, карточка абитуриента, форма договора
- [ ] Аудит-лог при смене статуса и подписании (`AuditService`)

## Статусы воронки

`new` → `contacted` → `contract_draft` → `contract_signed` → `enrolled` / `rejected`

## API-эндпоинты

| Метод | Путь | Описание |
|-------|------|----------|
| GET/POST | `/api/applicants` | Список и создание |
| PATCH | `/api/applicants/{id}/status` | Смена статуса |
| POST | `/api/applicants/{id}/contracts` | Создание договора |
| POST | `/api/contracts/{id}/sign` | Подписание |
| GET | `/api/managers/dashboard` | Dashboard менеджера |

## Критерий завершения

> [!success] Готово, когда
> Менеджер проводит абитуриента от «новый» до «договор подписан» через UI.
