# Модуль — Студенты

> [!info] Назначение
> Личный кабинет студента: расписание, профиль, уведомления. Read-only доступ к данным.

## Сущности

- **Student** — группа, курс, `user_id`
- **ScheduleSlot** (read) — пары студента через `schedule_slot_student`
- **Notification** — уведомления для студента

## API (prefix `/api/student`)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/student/schedule` | Пары на день/неделю |
| GET | `/api/student/profile` | Профиль: группа, курс |
| GET | `/api/student/notifications` | Список уведомлений |

## UI (student)

- **Расписание (сегодня)** — пары с временем, аудиторией, педагогом
- **Расписание (неделя)** — обзор на 7 дней
- **Профиль** — ФИО, группа, курс, контакты
- **Уведомления** — список с датами

## Адаптивность

> [!info] Mobile-first
> Студент чаще всего открывает кабинет с телефона. Карточки пар вместо таблиц на mobile (<768px). См. [[Этап 09 — Адаптивный UI]].

## Связь с абитуриентом

При зачислении (`enrolled`) абитуриент может быть связан со студентом (`Applicant` → `Student`).
